- [ ] **信号强度排序**：按 RSSI 降序排列，方便用户识别最近的设备
- [ ] **扫描超时与重试**：默认扫描超时后的空状态提示与重新搜索

- [ ] **建立连接**：选中设备后调用 SDK connect 接口

- [ ] **自动重连**：断连后自动重试，展示重连进度

## 设备模式管理

- [ ] **模式识别**：理解设备当前处于录音模式还是手势模式（启动默认录音模式）
- [ ] **模式切换**：通过按键单击（`0x0704`）在录音模式与手势模式之间切换
- [ ] **模式状态展示**：在界面中清晰展示当前设备模式
- [ ] **模式切换失败处理**：切换失败时（红灯闪 3 次）给出用户提示

## 动作事件处理

- [ ] **普通双击**（`0x0701`）：映射为「开始捕捉灵感」操作
- [ ] **HMM 手势识别**（`0x0702`）：解析 gesture_id（0 idle / 1 rotate_back / 2 rotate_front / 3 wave）
- [ ] **独立按键双击**（`0x0703`）：映射为用户预设的快捷操作
- [ ] **按键单击**（`0x0704`）：模式切换，同步更新 UI 状态
- [ ] **长按开始**：录音模式 → 开始录音；手势模式 → 开始 HMM 手势会话
- [ ] **长按松开**：结束录音或手势会话
- [ ] **手势映射配置**：在「戒指手势设置」界面支持自定义手势到操作的映射

## 录音与音频

- [ ] **录音开始/结束**：利用长按开始/松开控制录音生命周期
- [ ] **自动接收录音**：监听录音保存后设备主动上报的 `0x0505`（使用 `receiveAutoAudioFile`）
- [ ] **主动下载录音**：按文件索引调用 `downloadAudioFile(quick: true)` 下载历史录音
- [ ] **录音列表查询**：查询戒指上存储的录音文件列表
- [ ] **Speex → WAV 解码**：调用 SDK 解码链路（长度前缀 Speex → Ogg Speex → ffmpeg → WAV）
- [ ] **解码失败兜底**：保留原始 `.bin` 文件，解码失败时提示用户
- [ ] **清空录音**：`clearAudioFiles()`，破坏性操作需二次确认

## IMU 传感器

- [ ] **传感器上报开启**：切换至手势模式后调用 `startSensorReport()`（发送 `0x0601`）
- [ ] **批量数据接收**：解析 `0x0605` 批量六轴采样（时间戳 + 加速度 + 陀螺仪）
- [ ] **传感器上报关闭**：调用 `stopSensorReport()`（发送 `0x0603`）
- [ ] **数据用途**：可用于手势训练、动作识别优化等场景

## LED 状态反馈

- [ ] **录音中提示**：绿灯常亮（硬件行为，UI 可同步展示状态）
- [ ] **手势会话提示**：红灯常亮（硬件行为，UI 同步状态）
- [ ] **操作失败提示**：红灯闪烁 3 次（UI 展示错误 toast）
- [ ] **低电量拒绝操作**：电量低于 20% 时拒绝录音/手势（UI 展示低电量警告）
- [ ] **充电状态**：红灯呼吸 = 充电中，绿灯常亮 = 充满

## 协议边界理解（必须确认）

- [ ] BLE 一次通知 ≠ 一个完整协议包（需 `PacketStream` 重组，SDK 已封装）
- [ ] `0x0704` 收到 ≠ 模式切换成功（设备可能忙碌或处于长按收尾期）
- [ ] 长按松开后 250ms 内忽略短按（底层仍可能发 `0x0704`）
- [ ] 录音 `.bin` 是长度前缀 Speex 流，不能直接播放（需 SDK 解码为 WAV）
- [ ] 自动上报 `0x0505` 与 `downloadAudioFile()` 是两条不同入口，不能并发消费
- [ ] IMU `0x0605` 只在手势模式 + 已发送 `0x0601` 时才会上报
